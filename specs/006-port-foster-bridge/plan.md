# Implementation Plan: Port Reusable Features from foster-bridge

**Branch**: `006-port-foster-bridge` | **Date**: 2026-01-18 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/006-port-foster-bridge/spec.md`

## Summary

Port six reusable features from the foster-bridge repository back to the scaffold template: App-Level Error Boundary, Admin Auth Flow (login redirect, account settings), GitHub Actions for Firestore deployment, Structured Logging infrastructure, and Pre-commit Hook Setup. These features enhance developer experience and application stability.

## Technical Context

**Language/Version**: TypeScript 5.9.2, Node.js 18+
**Primary Dependencies**: NestJS 11.x, React 19.x, Firebase Admin SDK 13.x, Firebase Auth (client), Husky 9.x, Biome 2.3.11
**Storage**: Firebase Auth (identity + custom claims), Firestore (for logging endpoint - optional)
**Testing**: Vitest 4.0, Jest 30.x (for NestJS)
**Target Platform**: Web (Vite/React frontend), Node.js (NestJS backend), GitHub Actions (CI/CD)
**Project Type**: Nx monorepo (web + api + functions + libs)
**Performance Goals**: Admin redirect within 500ms of sign-in, Firestore deploy within 5 minutes of merge
**Constraints**: No external UI libraries (inline CSS only), must follow Feature Slice Design
**Scale/Scope**: 6 features, ~15 new files, 1 new library (libs/logging)

### Dependency Addition Review

**react-router-dom** is required for:
- Programmatic navigation (`useNavigate`)
- Declarative redirects (`<Navigate />`)
- Route protection

**Justification**: The current hash-based routing cannot support the admin auth flow requirements. React Router is the de facto standard for React routing and is already used in most React applications. This is not a premature addition but a necessary infrastructure upgrade.
