# Data Model: Admin User Seeding & CASL-Based RBAC

**Feature Branch**: `005-admin-rbac-casl`
**Date**: 2026-01-11

## Entities

### User

**Description**: Represents an authenticated user in the system. Identity stored in Firebase Auth, profile data stored in Firestore.

**Storage Locations**:
- **Firebase Auth**: Identity, credentials, custom claims (source of truth for role)
- **Firestore `/users/{uid}`**: Profile data, denormalized role for query convenience

#### Firebase Auth User Record

| Field | Type | Description | Source |
|-------|------|-------------|--------|
| uid | string | Unique identifier (auto-generated) | Firebase Auth |
| email | string | User's email address | Firebase Auth |
| emailVerified | boolean | Whether email is verified | Firebase Auth |
| displayName | string? | User's display name | Firebase Auth |
| disabled | boolean | Whether account is disabled | Firebase Auth |
| metadata.creationTime | string | ISO timestamp of account creation | Firebase Auth |
| customClaims.role | string | User's role (e.g., "admin") | Firebase Auth |

#### Firestore User Document (`/users/{uid}`)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| email | string | Yes | User's email (denormalized from Auth) |
| role | string | Yes | User's role (denormalized from Auth claims) |
| displayName | string | No | User's display name |
| createdAt | Timestamp | Yes | Server timestamp on creation |
| updatedAt | Timestamp | No | Server timestamp on update |

**Validation Rules**:
- `email`: Valid email format
- `role`: One of defined roles (initially just "admin")
- `createdAt`: Immutable after creation

**Indexes**: None required for initial scope (querying by uid via document path)

---

### Role

**Description**: Defines permission levels in the system. Stored as custom claim in Firebase Auth.

**Initial Roles**:

| Role | Description | Permissions |
|------|-------------|-------------|
| admin | Full system access | `manage all` (can perform any action on any subject) |

**Future Extensibility**:
```typescript
type Role = 'admin' | 'editor' | 'viewer';
```

---

### Ability (CASL)

**Description**: Runtime object defining what actions a user can perform on which subjects. Generated by `CaslAbilityFactory` based on user's role.

**Actions**:

| Action | Description |
|--------|-------------|
| read | View/read a resource |
| create | Create a new resource |
| update | Modify an existing resource |
| delete | Remove a resource |
| manage | Meta-action meaning "any action" |

**Subjects**:

| Subject | Description |
|---------|-------------|
| User | User entity |
| all | Meta-subject meaning "any subject" |

**Admin Ability Matrix**:

| Subject | read | create | update | delete |
|---------|------|--------|--------|--------|
| User | Yes | Yes | Yes | Yes |
| all | Yes | Yes | Yes | Yes |

## Relationships

```
Firebase Auth User (1) ──────────────── (1) Firestore User Document
         │                                        │
         │ uid                                    │ uid (document ID)
         │                                        │
         └── customClaims.role ────denormalized── role field
```

**Sync Strategy**:
- Firebase Auth custom claims are the **source of truth** for roles
- Firestore role field is denormalized for query convenience only
- Seeding script writes to both locations atomically
- Future role updates should update both Auth claims and Firestore document

## Security Rules (Firestore)

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users collection
    match /users/{userId} {
      // Allow read if authenticated user is reading their own document
      // or if user is admin
      allow read: if request.auth != null &&
        (request.auth.uid == userId || request.auth.token.role == 'admin');

      // Only admins can write to user documents
      allow write: if request.auth != null &&
        request.auth.token.role == 'admin';
    }
  }
}
```
