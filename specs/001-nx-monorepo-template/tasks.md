# Tasks: Nx Monorepo GitHub Template Repository

**Input**: Design documents from `/specs/001-nx-monorepo-template/`
**Prerequisites**: plan.md, spec.md, research.md, data-model.md, contracts/

**Organization**: Tasks are grouped by user story to enable independent implementation and testing of each story.

## Format: `[ID] [P?] [Story] Description`

- **[P]**: Can run in parallel (different files, no dependencies)
- **[Story]**: Which user story this task belongs to (e.g., US1, US2, US3)
- Include exact file paths in descriptions

## Path Conventions

This is an Nx monorepo with structure:
- `apps/web/` - React + Vite frontend
- `apps/api/` - NestJS backend
- `apps/functions/` - Firebase Functions
- `libs/shared-types/` - Shared TypeScript interfaces
- `tools/scripts/` - Setup and utility scripts

## Dependencies & Execution Order

### Phase Dependencies

- **Setup (Phase 1)**: No dependencies - can start immediately
- **Foundational (Phase 2)**: Depends on Setup completion - BLOCKS all user stories
- **User Story 3 (Phase 3)**: Depends on Foundational - Creates shared types needed by other stories
- **User Story 2 (Phase 4)**: Depends on US3 for types
- **User Story 4 (Phase 5)**: Depends on Foundational only - can parallel with US2
- **User Story 5 (Phase 6)**: Depends on US2 for API app
- **User Story 6 (Phase 7)**: Depends on US3 for types - can parallel with US5
- **User Story 1 (Phase 8)**: Depends on ALL previous phases (modifies all created files)
- **Polish (Phase 9)**: Depends on User Story 1 completion

### User Story Dependencies

```
Phase 1: Setup
    ↓
Phase 2: Foundational
    ↓
Phase 3: US3 (Shared Types) ←── Foundation for type safety
    ↓         ↘
Phase 4: US2   Phase 5: US4 (can parallel)
(Local Dev)    (CI/Biome)
    ↓              ↓
Phase 6: US5   Phase 7: US6 (can parallel)
(Cloud Run)    (Functions)
    ↓              ↓
    └──────┬───────┘
           ↓
Phase 8: US1 (Setup Script) ←── Must be last, modifies all files
           ↓
Phase 9: Polish
```

### Within Each User Story

- Types/models before services
- Services before controllers/UI
- Core implementation before integration
- Verification task at end of each story

### Parallel Opportunities

- All Setup tasks marked [P] can run in parallel
- All Foundational tasks marked [P] can run in parallel (within Phase 2)
- US4 (Biome/CI) can run in parallel with US2 after US3 completes
- US5 (Cloud Run) and US6 (Functions) can run in parallel
- Within each phase: all tasks marked [P] can run in parallel
